# お絵描きライブ v0.4.1

## 追加機能
- 参加者は参加時に自動で目隠し状態
- 管理画面から参加者ごとに「公開する／目隠しする」切替
- 参加者名または「アップ表示」ボタンで、その参加者をOBS全面表示
- 「通常表示に戻す」で1人・2人・4人レイアウトへ復帰
- 目隠しはOBS上のカバー表示のみ。描画データの送受信は継続

## v0.2.0 シンクロイラスト機能追加
- 新しい表示モード「シンクロ表示（合体イラスト）」を追加。管理画面の「表示モード」からグリッド表示⇔シンクロ表示を切替できます。
- 管理画面の参加者カードに「パーツ名」欄を追加。胴体・手足・顔など自由入力で担当パーツを割り当てられます（`participants/slotN/part`）。
- シンクロ表示中は、各スロットの絵を同じ1280×720キャンバスに重ねて1枚の合成イラストとして表示します。重なる順番はスロット番号順（slot1が一番下、slot4が一番上）固定です。
- 個別の「目隠し／公開」はそのまま利用可能。目隠し中のパーツは合成に含まれず、公開した瞬間にそのパーツだけが合成イラストに重なります。→「一人ずつ順番に公開して最後に全体を見せる」演出に対応。
- 合成キャンバスの下に、パーツ名と公開状況（公開済み／準備中）を示すステータスバッジを表示します。
- 参加者ページ（index.html）にも自分の担当パーツ名を表示するようにしました（位置合わせ用のガイド画像などは今回は導入していません。各自の感覚で描く方式）。
- ポジションのガイド表示は非搭載です。パーツ同士の位置関係は、担当分けと各自の描き方で調整してください。

## v0.2.2 シンクロ合成の重大バグ修正
- シンクロ表示で2人以上が同時に公開されているとき、スロット番号が一番大きい（最前面の）参加者の絵しか見えず、それより下のレイヤーの絵が白で塗りつぶされて消えてしまう不具合を修正しました。
- 原因：各参加者の裏側キャンバス（オフスクリーン）を不透明な白でクリアしていたため、重ね合わせ時に下のレイヤーを覆い隠していました。オフスクリーン側は透明でクリアするように修正し、最終的な合成キャンバスのみ白背景にしています。
- これにより、複数人を順番に公開していったときに、それぞれの絵がちゃんと重なって残るようになりました。

## v0.3.0 共有キャンバスモード・画像保存機能を追加
- 新しい表示モード「共有キャンバス（みんなで1枚）」を追加。管理画面の「表示モード」から選べます。
- 共有キャンバス中は、参加者全員が同じ1枚のキャンバスにリアルタイムで一緒に描き込めます（`drawingRooms/{room}/shared`）。参加者ページ側にも他の人の線がリアルタイムで反映されます。
- 「絵を消す」は共有キャンバスでは個別にはできず、管理画面の「共有キャンバスを全消し」で全員分をまとめて一括で消す仕様にしました。
- 参加者ページ（消しゴムボタンの隣）と管理画面の両方に「画像を保存」ボタンを追加。押すとPNG画像としてダウンロードできます。
  - 参加者ページ：今自分の画面に表示されているキャンバスをそのまま保存（共有モード中はみんなの絵、それ以外は自分の絵）。
  - 管理画面：シンクロ表示中は「公開する」済みのパーツを合成した完成イラストを、共有キャンバス中は共有キャンバスをまとめて保存。グリッド表示中は1枚に定まらないため保存不可（その旨アラート表示）。
- 参加者ページの受信処理に、共有キャンバスで他の人の線を再生する際、クリア前の古い線が紛れ込まないようにする対策を追加しました（現在のclearVersionを先に確定してから過去の線を再生）。

## v0.3.1 「参加枠がいっぱいです」の根本原因を修正
- 原因：「全員の絵を全消し」ボタンが、参加者がいない空きスロットにも`clearVersion`・`clearAt`を書き込んでしまっていました。一度でも押すと、空きスロットが`{clearVersion:1, clearAt:...}`という「participantIdが無いだけのデータ」を持つ状態になり、参加処理の`cur===null`という判定が「空きではない」と誤判定していました（admin.html側の参加者一覧はparticipantIdの有無だけを見るので「空き」と正しく表示されるため、発見しづらいバグでした）。
- 修正1：`index.html`の参加処理を、`cur===null`だけでなく「participantIdが無ければ空き扱い」にするよう緩和しました。
- 修正2：`admin.html`の「全員の絵を全消し」は、参加者がいるスロットにだけ処理するようにしました（空きスロットを汚さない）。
- 注意：この修正はコード側の再発防止です。既にデータが汚れてしまった部屋（例：abo-room）は、管理画面の「部屋を全部リセット」を一度押すか、Firebaseコンソールで空きスロットの`clearVersion`・`clearAt`を手動削除してください。

## v0.3.2 OBS側の「線が消える」レース条件を修正
- 原因：OBS表示（obs.html）は各スロットの「誰が参加しているか（participantId/occupied）」と「線データ（strokes）」を別々に読み込んでいましたが、コードが「参加者情報が先に届く」前提になっていました。実際のFirebase通信ではどちらが先に届くか保証されないため、線データの方が先に届くと、その線を黙って捨てて二度と描画しないバグがありました。
- 特にシンクロ表示に切り替えるたびに全スロットの接続を作り直すため、このタイミング次第で特定の参加者だけ絵が表示されない・グリッドに戻しても復活しない、という現象が起きていました。
- 修正：線を読み込むタイミングでの参加者情報チェックを廃止し、表示するかどうかは別の場所（CSSの表示切り替え・シンクロの合成処理）で正しく判定するようにしました。
- 実際に「参加者情報の読み込みが線データより遅れる」状況を再現するテストを行い、修正前は再現・修正後は解消することを確認済みです。

## v0.3.3 共有キャンバスへのモード切替に確認ダイアログを追加
- 「共有キャンバス」への切替・「共有キャンバス」からの切替の両方で、切り替える前に確認ダイアログが出るようにしました。「はい」を選ぶと、参加者全員のプライベート分の絵＋共有キャンバスの絵をすべて消してからモードを切り替えます。「いいえ」を選ぶとモード切替自体をキャンセルします（絵もモードもそのまま）。
- グリッド⇔シンクロの切替（共有キャンバスが絡まない場合）は今まで通り確認なしで即切り替わります。
- 共有キャンバスの参加人数上限は引き続き4人です（参加枠と共通のため）。

## v0.4.0 表示比率の修正・「ひとつ戻る」ボタンを追加
- OBS表示の「2人」表示で画面が縦長に潰れてしまう不具合を修正しました。CSS Gridの1frによる引き伸ばしをやめ、1人・2人・4人どの表示でも各枠が16:9を保ったまま最大サイズになるようJSでセルサイズを計算するようにしました（実測でどのレイアウトでも比率16:9を確認済み）。フォーカス表示（アップ表示）は今まで通り画面いっぱいに表示されます。
- 参加者ページに「↩️ ひとつ戻る」ボタンを追加しました。ペンを置いてから離すまでの一筆を1操作として記録し、押すたびに直近の一筆を取り消せます（送信のバッチ分割に関わらず、一筆＝1回のUndo単位になるようにしています）。
- 取り消した線は、自分の画面だけでなくOBS表示・シンクロ合成・共有キャンバス・共有モードで一緒に描いている他の参加者の画面にもリアルタイムで反映されます（線を1本だけ消すことはcanvasの仕組み上できないため、削除を検知したら現在残っている線を全部読み直して描き直す方式にしています）。
- 「自分の絵を全消し」やモード切替をした場合は、Undo履歴もあわせてリセットされます。

## v0.4.1 表示人数（1人/2人/4人）切替時に絵を全消去
- 管理画面で「1人」「2人」「4人」表示を切り替えるとき、切り替え前に参加者全員の絵を消してから切り替えるようにしました（確認ダイアログなしで即実行されます）。
